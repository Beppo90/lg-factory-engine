## 8 REGLAS DE DISEÑO

### REGLA 1 — TERCER NIVEL DE TEXTOS PARALELOS
Los textos del examen final son un TERCER nivel de paralelismo:
- **Nivel 1 (Guía):** Carlos Ramírez — email + phone call
- **Nivel 2 (PM-4.2):** Laura Méndez — memo + Miguel Torres email + IT voicemail
- **Nivel 3 (PM-2.10):** Personajes NUEVOS, textos NUEVOS — que no aparecen ni en la guía ni en el cuestionario

**Restricción estricta:** NO reutilizar Carlos, Laura, Miguel, ni el voicemail del IT Department. Nuevo personaje, nuevo texto, mismo universo (DevCore Solutions).

### REGLA 2 — 100 PUNTOS, 6 SECCIONES

| Sección | Habilidad | Puntos | Bloom |
|---------|-----------|--------|-------|
| 1. Reading Comprehension | Receptiva | 15 | Remember / Understand / Analyze |
| 2. Listening Comprehension | Receptiva | 15 | Remember / Understand / Analyze |
| 3. Vocabulary in Context | Productiva | 15 | Apply / Analyze / Evaluate |
| 4. Grammar & Structure | Productiva | 20 | Apply / Analyze / Evaluate |
| 5. Writing Task | Productiva | 20 | Apply / Create |
| 6. Integrated Task | Productiva | 15 | Analyze / Evaluate / Create |
| **Total** | | **100** | |

### REGLA 3 — SCAFFOLDING MODERADO (NO ALTO)
A diferencia de PM-4.2 que tiene Language Bank + Skeleton Structure completos, el examen final reduce el scaffolding:

- **Writing Task (Sección 5):** Language Bank reducido (4-5 chunks, no 8). SIN Skeleton Structure. El aprendiz debe recordar el formato del email.
- **Vocabulary (Sección 3):** Word Banks reducidos o ausentes en algunas tasks.
- **Grammar (Sección 4):** Formulas proporcionadas pero ejemplos reducidos.

**Principio:** El examen final evalúa INDEPENDENCIA, no solo transferencia con soporte. El aprendiz debe demostrar que puede producir sin todos los andamiajes.

### REGLA 4 — SECCIÓN 6: INTEGRATED TASK
La Sección 6 es la diferencia clave con PM-4.2. Es una tarea integradora que combina TODAS las habilidades en un solo escenario:

- El aprendiz recibe un escenario situacional complejo
- Debe: leer un breve texto → tomar una decisión → escribir una justificación → usar vocabulario técnico → usar grammar targets correctamente
- Evalúa transferencia real, no recall aislado

**Formato de la Sección 6:**
- 1 escenario (3-4 líneas)
- 3 preguntas que requieren producción escrita
- Evaluación con rúbrica (no multiple choice)

### REGLA 5 — NIVEL CEFR ESTRICTO (IGUAL QUE PM-4.2)
- A1.1-A1.2: Present simple, short sentences (≤15 words)
- Instrucciones bilingües
- NO vocabulary o structures fuera de la guía
- Tiempo límite: 120 minutos (más que PM-4.2 por ser más largo)

### REGLA 6 — COHERENCIA CON PM-4.2
El examen final DEBE ser coherente con el cuestionario pero NO redundante:
- Mismo universo narrativo (DevCore Solutions)
- Personajes diferentes a los del cuestionario
- Textos de diferente género (si PM-4.2 usó memo, PM-2.10 puede usar announcement o report)
- Tasks de mayor complejidad (más HOTS, menos scaffolding)
- Cubre gaps del cuestionario (si PM-4.2 no evaluó pronunciación en contexto, PM-2.10 puede incluirlo implícitamente)

### REGLA 7 — ANSWER KEY COMPLETO (IGUAL QUE PM-4.2)
- Respuestas correctas con distribución de puntos
- Alternativas aceptables
- Rúbrica detallada para Writing y Integrated Task
- Sample responses para tareas abiertas
- Assessment Summary table
- Distribución Bloom

### REGLA 8 — ZERO META-TALK (IGUAL QUE PM-4.2)
El output es el examen LISTO PARA IMPRIMIR. No incluye:
- ❌ Explicaciones de diseño
- ❌ Notas metodológicas en el cuerpo del examen
- ❌ Comentarios sobre Bloom o SIOP

El único contenido meta es el TTS Note (Listening) y el Answer Key (al final, separado).

## ESTRUCTURA DETALLADA POR SECCIÓN

### Section 1 — READING COMPREHENSION (15 pts)
- Texto ESP nuevo (≈300 palabras — más largo que PM-4.2)
- Género paralelo a la guía Y al cuestionario
- Task A: Main Purpose (3 pts, multiple choice)
- Task B: Information Extraction (4 pts, tabla con 4 ítems)
- Task C: True/False/Not Given (4 pts, 4 statements)
- Task D: Inference (4 pts, 2 preguntas de inferencia con justificación)

### Section 2 — LISTENING COMPREHENSION (15 pts)
- Script nuevo (≈180 palabras — más largo que PM-4.2)
- Género paralelo diferente
- TTS Note para el instructor
- Task A: Multiple Choice (4 pts, 4 questions)
- Task B: Gap Fill con Word Bank (6 pts, 6 blanks + 2 distractors)
- Task C: Short Answer (5 pts, 5 preguntas con respuesta corta)

### Section 3 — VOCABULARY IN CONTEXT (15 pts)
- Task A: Apply (5 pts) — Matching components with functions (5 items)
- Task B: Analyze (5 pts) — Odd One Out (2 items × 2.5 pts) + justification
- Task C: Evaluate (5 pts) — Scenario-based vocabulary selection + justification

### Section 4 — GRAMMAR & STRUCTURE (20 pts)
- Task A: Apply (6 pts) — Fill in the blank with correct structure (6 items × 1 pt)
- Task B: Analyze (8 pts) — Error Correction: find error + correct + explain rule (4 items × 2 pts)
- Task C: Evaluate (6 pts) — Write a short paragraph (4-5 sentences) describing a workstation using ALL grammar targets (is/has/demonstratives/comparisons)

### Section 5 — WRITING TASK (20 pts)
- Escenario situacional
- Language Bank reducido (4-5 chunks)
- SIN Skeleton Structure (el aprendiz recuerda el formato)
- Rúbrica visible: Format (4) + Grammar Accuracy (6) + Vocabulary Use (6) + Communicative Clarity (4)

### Section 6 — INTEGRATED TASK (15 pts)
- Escenario complejo (combina lectura + decisión + escritura + vocab + grammar)
- 3 preguntas productivas
- Rúbrica: Analysis (5) + Justification (5) + Language Use (5)
